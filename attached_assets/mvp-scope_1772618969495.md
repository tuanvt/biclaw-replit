# BiClaw — MVP Product Scope

## MVP Definition

The minimum product that lets a user sign up, get a running BI assistant, connect their business data sources (Shopify, Stripe, Google Analytics, Facebook Ads, Google Sheets, or any web dashboard), and receive a proactive morning brief — all within their existing chat app.

**The connector IS the product. Not an afterthought.**

---

## User Flow (End-to-End) — Conversational Onboarding

**There is no signup form.** The entire onboarding happens inside a conversation with BiClaw.

See `docs/product/conversational-onboarding.md` for the full architecture and detailed conversation flows.

```
1. User visits biclaw.app
2. Chat widget auto-opens: "Hi! I'm BiClaw. What do you sell?"
   (or user clicks "Try it on WhatsApp" → same flow on WhatsApp)
3. User replies — e.g., "I sell phone cases on Shopify"
4. BiClaw qualifies: "Do you also run Facebook Ads?"
5. BiClaw offers live demo: "Share your Shopify URL — I'll show you what I can do"
6. User shares URL → BiClaw analyzes their store data RIGHT THERE
7. BiClaw shows the visitor their real numbers (or a sample brief)
8. Visitor is hooked → "Want me to send you this every morning on WhatsApp?"
9. User says yes → provides WhatsApp number + preferred brief time
10. BiClaw provisions a dedicated instance in the background (~60 seconds)
11. User receives a welcome message on WhatsApp from their own BiClaw
12. BiClaw continues onboarding on WhatsApp: connects remaining sources
13. Next morning: BiClaw sends a summary combining all connected sources
14. User can ask follow-up questions anytime via chat

No email. No password. No "choose your plan." Just a conversation.
```

### Two-Agent Model (Multi-Agent Gateway)

| Phase | Agent | Purpose |
|-------|-------|---------|
| Demo (steps 2-8) | Shared "demo" agent | Handles all landing page chats via OpenClaw's built-in WebChat. Uses BiClaw's LLM key (GPT-4o-mini). Sessions expire after 15 min of inactivity. |
| Conversion (steps 9-14) | Dedicated per-user agent | Private agent with isolated workspace + Docker sandbox. User's own LLM key. Persistent. |

Both run on the same OpenClaw gateway (multi-agent routing). The transition is seamless — the provisioning service creates a new agent workspace and routing binding. No separate containers needed.

## Core Features (MVP Only)

### F0: Shared Demo Instance (Landing Page Chat)

**What:** A live BiClaw chat widget on the landing page powered by a shared OpenClaw demo instance. Visitors can talk to BiClaw, connect their data, and see real analysis — all before any signup.

**Details:**
- One always-running OpenClaw instance with BiClaw SKILL_DEMO.md (strict sales funnel persona)
- Handles all landing page chat widget conversations via session IDs
- Uses BiClaw's own LLM API key (~$0.02-0.10 per conversation depending on model: GPT-4o-mini ~$0.02, GPT-5-mini ~$0.04, Claude Haiku ~$0.10)
- Can read Shopify store data, show sample briefs — ONLY BiClaw-related tasks
- Sessions expire after 15 min of inactivity
- No credentials stored long-term in the demo instance
- **5-layer guardrail pipeline** enforces strict scope (see `docs/product/conversational-onboarding.md`)

**Acceptance criteria:**
- [x] Chat widget on landing page connects to shared demo instance via WebSocket
- [x] Multiple simultaneous visitors can chat without crosstalk
- [x] BiClaw qualifies visitor (what they sell, what platforms they use)
- [x] BiClaw can analyze a Shopify store when visitor shares URL/token
- [x] BiClaw can show a sample morning brief with demo data
- [x] Session cleanup after 15 min inactivity
- [ ] Quick reply buttons work ("I sell on Shopify" / "I sell on Shopee" / "Just browsing")

**Guardrail acceptance criteria:**
- [x] Off-topic messages (code requests, general knowledge, creative writing) are blocked pre-LLM with a canned redirect — no LLM tokens spent
- [x] Hard limit: 30 messages per session, 2000 chars per message
- [x] Hard limit: 3 sessions per IP per hour, 1 concurrent session per IP
- [x] Prompt injection / jailbreak attempts are caught (post-LLM response check blocks code blocks, off-topic outputs)
- [x] Conversation state machine tracks phase (GREETING → QUALIFYING → DEMOING → CONVERTING → STALLED)
- [x] After 5+ consecutive off-topic messages → session moves to STALLED with "try ChatGPT" message
- [x] Demo instance uses SKILL_DEMO.md (strict persona), NOT the full SKILL.md
- [x] All guardrail events are logged for analytics (blocked_pre_llm, off_topic, jailbreak_attempt, stalled)
- [x] Average LLM cost per demo session < $0.15

### F1: Conversational Provisioning (Demo → Dedicated)

**What:** When a visitor says "set me up," BiClaw provisions a dedicated instance and transitions them to their chosen channel — all inside the conversation.

**Details:**

- Automated Docker container provisioning in background
- Pre-configured with BiClaw SKILL.md + tools
- Data sources from demo session migrate to dedicated instance
- Conversation context transferred
- Welcome message sent on chosen channel within 60 seconds

**Acceptance criteria:**
- [x] Visitor can go from demo chat to dedicated instance without any form
- [x] Instance is live within 60 seconds of commitment
- [x] Welcome message arrives on chosen channel
- [x] Shopify tokens collected during demo migrate to dedicated instance
- [x] BiClaw skill is functional immediately on dedicated instance
- [x] Demo session cleaned up after migration

### F2: 6-Layer Data Connectors

**What:** User connects their business data sources. BiClaw supports 6 connection methods — all available from day 1. Leverages every OpenClaw capability.

**See `docs/architecture/connector-strategy.md` for full architecture detail.**

#### Layer 1: Webhooks (PUSH — real-time events, free via OpenClaw)
- **Shopify webhooks** — instant order alerts, refund warnings, product updates
- **Stripe webhooks** — payment succeeded/failed, subscription changes
- **WooCommerce webhooks** — order events
- Agent receives events instantly and alerts the user in real time
- Webhook data accumulates throughout the day for morning brief (no API call needed)
- Setup: user adds a webhook URL in their Shopify/Stripe admin (guided by BiClaw)

#### Layer 2: Gmail Pub/Sub (PUSH — email parsing, free via OpenClaw)
- **Shopee order emails** — parsed for order total, product, buyer
- **Lazada order emails** — same parsing
- **TikTok Shop order emails** — same parsing
- **Bank transfer notifications** — parsed for amount, sender
- **Stripe/PayPal receipt emails** — backup data source
- **Shipping notifications** — delivery status tracking
- Zero friction: user grants Gmail read access, BiClaw monitors automatically
- **KEY: This is the connector for platforms without APIs.** A Shopee seller just grants Gmail access — no login sharing, no API key, no browser.

#### Layer 3: Direct API Tools (PULL — smart local tools, ~80-100 lines each)
- **Shopify** — user creates a Custom App → provides access token
- **Stripe** — user creates a restricted API key (read-only)
- **Google Analytics** — user grants access to BiClaw service account
- Tools run locally in the instance, call APIs directly (no backend)

#### Layer 4: Browser Control (PULL — free via OpenClaw, zero code)
- **Facebook/Meta Ads** — agent opens Ads Manager, reads campaign data
- **Google Ads** — agent opens Ads dashboard, reads performance
- **Shopee Seller Center** — agent opens dashboard for detailed metrics
- **Lazada Seller Center** — same
- **Any web dashboard** — agent navigates and reads

#### Layer 5: MCP Servers (PULL — ecosystem, zero build)
- Community MCP servers for Shopify, Stripe, GA (if OpenClaw supports MCP)

#### Layer 6: Google Sheets (PULL — universal fallback)
- User shares a Google Sheet URL for custom data

**The agent picks the best method automatically per platform.** Shopify: webhooks + API. Shopee: Gmail + browser. Facebook Ads: browser. The user doesn't choose the layer — BiClaw recommends the lowest-friction option.

**Acceptance criteria:**
- [ ] Shopify webhooks receive and parse order/refund events in real time
- [ ] Stripe webhooks receive and parse payment/subscription events
- [ ] Gmail Pub/Sub monitors for Shopee/Lazada order emails and parses them
- [ ] Gmail Pub/Sub monitors for bank transfer notifications
- [x] Shopify API tool connects and reads orders/revenue (for morning brief depth)
- [x] Stripe API tool connects and reads payments/revenue
- [x] Google Analytics API tool connects and reads traffic data
- [x] Browser can log into and read data from Facebook Ads, Shopee, Lazada
- [x] Google Sheets connection works (share URL in chat)
- [ ] Webhook data accumulates and feeds into morning brief automatically
- [ ] Gmail-parsed data feeds into morning brief automatically
- [x] User can connect multiple sources across multiple layers
- [x] Morning brief combines data from all connected layers

### F3: On-Demand Data Analysis

**What:** User asks a question about their data in natural language. BiClaw answers with numbers and context.

**Example conversations:**
```
User: How was revenue yesterday?
BiClaw: Yesterday's revenue was 12.4M VND — down 8% from the day before
        (13.5M), but up 12% vs. same day last week (11.1M).
        Orders were 47 (vs. 52 day before). Average order value
        held steady at ~264K VND.

User: What's my best day this month?
BiClaw: Feb 8 was your best day — 18.2M VND revenue from 67 orders.
        That was a Saturday. Your Saturdays average 15.1M this month
        vs. 11.8M on weekdays.

User: Compare this week to last week
BiClaw: This week (Mon-today): 52.3M VND revenue, 198 orders
        Last week (same days): 48.1M VND revenue, 182 orders
        You're up 8.7% in revenue and 8.8% in orders.
        Ad spend is flat (3.2M vs 3.1M), so your ROAS improved
        from 15.5x to 16.3x.
```

**Acceptance criteria:**
- [x] Answers natural language questions about sheet data
- [x] Provides context (day-over-day, week-over-week comparisons)
- [x] Handles Vietnamese currency formatting (VND)
- [x] Responds within 10 seconds

## Explicitly Out of Scope (MVP)

| Feature | Why deferred |
|---------|-------------|
| Multi-agent setup | Overkill for single-user BI |
| Canvas / interactive dashboards | Chat-based delivery is the MVP format |
| Voice queries | Nice-to-have, not essential |
| White-label / agency features | Post-PMF |
| Custom anomaly rules | Default statistical rules first |
| Multiple chart types | Bar + line is enough |
| PDF report export | Chat delivery is the MVP format |
| Billing system | Manual / Stripe checkout link for first users |
| OAuth flows for connectors | API keys + browser login + Gmail cover MVP needs |

## Onboarding Flow (Conversational — No Forms)

The full conversation flow is documented in `docs/product/conversational-onboarding.md`. Below is the condensed version.

### Phase 1: Hook (Landing Page Chat Widget)
```
[Visitor lands on biclaw.app]
  → Chat widget auto-opens after 5 seconds

BiClaw: Hi! I'm BiClaw — a business data analyst that
        lives in your WhatsApp.

        I can connect to your Shopify, Facebook Ads,
        Shopee, Lazada, or Google Sheets — and send you
        a morning brief every day.

        What do you sell?

        [ I sell on Shopify ]  [ I sell on Shopee ]  [ Just browsing ]
```

### Phase 2: Qualify + Live Demo
```
Visitor: I sell phone cases on Shopify

BiClaw: Nice! Do you also run Facebook Ads?

Visitor: Yeah

BiClaw: Got it — Shopify + Facebook Ads. I can combine both
        into one daily morning brief. Want me to take a look
        at your Shopify store right now?

Visitor: Sure — mystore.myshopify.com

BiClaw: [Analyzes public storefront or asks for API token]
        [Shows real data analysis or sample brief]

        Here's what your morning brief would look like:
        ──────────────────────────────
        ☀️ CaseCraft daily brief:
        🛒 Revenue: 12.4M VND (↓8%)
        📱 Ad spend: 520K · ROAS: 23.8x
        ⚠️ Orders below average
        ──────────────────────────────

        Want to get this every morning on WhatsApp?
```

### Phase 3: Conversion (No Form)
```
Visitor: Yes, set me up

BiClaw: I need two things:
        1. WhatsApp number?
        2. Morning brief time? (default: 8 AM Saigon time)

Visitor: +84 xxx xxx xxxx, 8am is good

BiClaw: Setting up your private BiClaw instance...

        ✓ Private instance created
        ✓ Shopify connected
        ✓ WhatsApp configured
        ✓ Morning brief: 8:00 AM daily

        Check your WhatsApp — I just sent you a welcome message!
```

### Phase 4: Continue on WhatsApp (Dedicated Instance)
```
[WhatsApp]

BiClaw: Hi! I'm your BiClaw — now running on your private
        instance. Everything from our website chat is set up.

        ✓ Shopify: CaseCraft connected
        ✓ Morning brief: 8 AM daily

        Want to also connect your Facebook Ads? That way
        tomorrow's brief will include ROAS and ad performance.

Visitor: Yes

BiClaw: [Walks through Facebook connection via browser or API]

[Next morning, 8:00 AM]

BiClaw: ☀️ Good morning! Here's your brief...
```

### Edge Cases

| Scenario | BiClaw's Response |
|----------|-------------------|
| Visitor doesn't want to share data during demo | Shows a sample brief with demo data from a test store |
| Visitor says "just browsing" | Gives 30-second elevator pitch + sample brief image, invites them back anytime |
| Visitor provides token then ghosts | Session expires after 15 min of inactivity, all temp data wiped |
| Visitor wants Telegram, not WhatsApp | Same flow, asks for Telegram username instead |
| High traffic (many simultaneous chats) | Shared instance handles concurrent sessions; rate limit 3 per IP |

## Quality Bar

- **Response time:** <10 seconds for data queries
- **Morning brief reliability:** 99%+ delivery rate
- **Data accuracy:** Numbers must match the source exactly (no hallucination)
- **Chart quality:** Clean, branded, readable on mobile screens
- **Demo chat engagement:** >30% of landing page visitors send at least 1 message
- **Demo-to-conversion rate:** >8% of chat conversations result in a dedicated instance
- **Onboarding completion:** >60% of converted users connect at least 1 data source
- **Provisioning speed:** Demo → dedicated instance transition in <60 seconds
